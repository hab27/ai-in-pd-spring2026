# MiniClaw Starter RAG

This is a starter RAG package for **MP3 Part B**. Use it if your MP2 RAG
isn't working or if you'd rather start fresh from a known-good baseline.
If your MP2 RAG **is** working, use yours — that's better practice.

The corpus is the same set of fictional ACME Robotics documents you saw
in MP2 Part B (15 files spanning manufacturing, materials, prior products,
vendors, and project status).

## Setup

```bash
# 1. From the repo root, build the ChromaDB index (one time, ~30 seconds).
python "MP3/Part B/miniclaw_starter_rag/build_index.py"

# 2. Smoke-test the query helper.
python "MP3/Part B/miniclaw_starter_rag/query_miniclaw_rag.py"
```

You should see something like:

```
3 chunks retrieved. Top match: ACME-MFG-002 — FilaTech PolyPro PLA Technical Summary ...
```

## How it fits the assignment

The function `query_miniclaw_rag(question, n_results=3) -> dict` is what
you wrap as a tool in your MCP server (`mcp_server_starter/server.py`).
The MCP server exposes the same function over the wire so an AI host
(Copilot agent mode, Claude Desktop, Cursor) can call it.

**The principle is identical to MP3 Part A Section 3** — only the
transport changes. In Part A you wrapped a Python function with a JSON
schema and the OpenAI SDK called it. Here, you wrap it with the MCP
decorator and a host calls it. Same idea, different protocol.

## Files

| File | Purpose |
|------|---------|
| `acme_miniclaw_corpus.py` | Loads the 15 ACME documents from `MP2/Part B/corpus/` |
| `build_index.py`          | One-time ChromaDB build (run once) |
| `query_miniclaw_rag.py`   | The function you'll expose as an MCP tool |
| `chroma_db/`              | Generated by `build_index.py` — do NOT edit by hand |
